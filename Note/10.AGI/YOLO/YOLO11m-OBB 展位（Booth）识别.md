# YOLO11m-OBB 展位（Booth）识别
## 增量训练优化方案（小样本·400张混合数据集）

---

## 一、项目背景与目标
- **模型**：YOLO11m-OBB（旋转目标检测，专用于展位 Booth 检测）
- **任务**：展位目标识别（带旋转框 OBB）
- **数据规模**：新旧图片共 **400 张**（小样本增量训练）
- **训练分辨率**：`imgsz=1024`
- **核心目标**
  - 在原有模型基础上**增量学习新类别/新场景**
  - 避免**灾难性遗忘**，保留旧类别精度
  - 解决**图片分辨率不统一、标注标准不一致、数据分布不一致**问题
  - 在小样本下保证模型稳定收敛、泛化性强

---

## 二、核心问题梳理
1. **增量训练风险**
   - 只训新数据 → 旧类别精度暴跌（灾难性遗忘）
   - 标注/分布不一致 → 训练震荡、不收敛、OBB 框不准
2. **数据问题**
   - 总数据仅 400 张，对噪声、标注错误、分布差异**极度敏感**
   - 原始图片分辨率不统一，训练分辨率固定 1024
3. **模型特性**
   - OBB 对**几何形状、宽高比、旋转角度、标注框精度**更敏感
   - 高分辨率（1024）对显存、学习率、批次大小要求更高

---

## 三、数据集统一规范（最关键）
### 3.1 标注标准完全统一
所有新旧数据必须遵守**同一套标注规则**，建议人工全量校验。

1. **类别名与类别 ID 统一**
   - 提供唯一 `classes.txt` / `booth_obb.yaml` 类别清单
   - 禁止：别名、大小写混乱、ID 错乱、错别字
   - 格式示例：
     ```
     0: booth
     1: advertising_board
     ...
     ```

2. **OBB 旋转框规则统一**
   - 框的松紧：**紧贴目标，少留背景**（统一标准）
   - 遮挡规则：可见部分≥50% 才标，否则不标
   - 旋转角度体系统一：
     - 统一使用 `0~180°` 或 `-90~90°`，新旧数据必须一致
   - 禁止：同一目标有时标水平框、有时标旋转框

3. **标注完整性统一**
   - 每张图中**所有目标必须全部标注，不漏标**
   - 漏标会直接导致模型学习“假负样本”，引发漏检

4. **标注文件格式统一**
   - OBB 格式：`class_index x1 y1 x2 y2 x3 y3 x4 y4`（四点坐标）
   - 坐标归一化/像素级必须与模型要求一致

---

### 3.2 数据分布统一（分辨率不统一解决方案）
训练设置 `imgsz=1024`，原始图分辨率杂乱，**必须使用等比缩放+填充**。

1. **统一处理方式：Letterbox**
   - 按比例缩放到**最长边=1024**
   - 短边用黑边填充到 1024×1024
   - 标注框坐标同步缩放，**不拉伸、不变形**

2. **YOLO 自动实现（无需手动预处理）**
   - 训练时启用：
     ```python
     imgsz=1024,
     letterbox=True,   # 默认开启，强烈建议显式指定
     rect=False        # 小样本关闭矩形训练，保证批次尺寸一致
     ```

3. **基础属性对齐**
   - 所有图片统一亮度、对比度范围（避免白天/夜间差异过大）
   - 拍摄角度、距离尽量接近原有数据集
   - 不允许新数据全是极端场景（如全黑、全模糊、极端俯视）

4. **新旧数据比例**
   - 推荐：**旧数据 : 新数据 = 1:1 或 3:2**
   - 禁止：只训新数据 或 新数据占比过高（如 350:50）

## 四、增量训练超参数方案（YOLO11m-OBB 专用）
### 4.1 基础训练配置
```python
model = YOLO('yolo11m-obb.pt')

results = model.train(
    data='booth_obb.yaml',
    imgsz=1024,
    letterbox=True,
    rect=False,
    
    epochs=100,          # 小样本不用过多轮，防止过拟合
    patience=15,         # 早停
    
    batch=8,             # 1024 分辨率建议 batch=4~8
    # 显存不足时使用：
    # batch=4, accumulate=2
    
    device=0,
    workers=4,
    seed=42,
)
```

---

### 4.2 学习率（增量训练核心）
原则：**小学习率，防止覆盖旧知识**。

```python
lr0=1e-4,         # 初始学习率（比从头训练小 10~100 倍）
lrf=0.01,         # 学习率衰减
warmup_epochs=3,  # 热身
momentum=0.937,
weight_decay=0.0005,
```

### 4.3 分层训练（防止遗忘）
先冻骨干，只训检测头，再解冻微调：
```python
freeze=10,  # 先冻结前10层骨干网络，训练头部
```

---

### 4.4 OBB 专用数据增强（小样本安全版）
OBB 对透视、剪切非常敏感，**只开温和增强**。

```python
hsv_h=0.015,
hsv_s=0.7,
hsv_v=0.4,

degrees=10.0,    # 仅小角度旋转，适配展位
translate=0.1,
scale=0.5,

shear=0.0,       # 关闭剪切
perspective=0.0, # 关闭透视
flipud=0.0,      # 垂直翻转按需关闭
fliplr=0.5,      # 水平翻转可开
```

---

### 4.5 锚框优化（1024 分辨率必做）
基于数据集自动计算适配 OBB 的锚框：
```python
from ultralytics.data.utils import check_anchors
check_anchors(data, imgsz=1024, model=model, thr=4.0)
```

## 五、增量训练流程（标准步骤）
1. **数据合并**
   - 新旧图片、标注放到同一训练集
   - 统一类别、统一标注规则、统一 OBB 格式
2. **数据清洗与校验**
   - 检查错标、漏标、重复标注
   - 随机可视化 20~50 张，确认 OBB 框准确
3. **分辨率统一**
   - 依靠 YOLO letterbox 自动等比缩放+填充
4. **第一阶段训练（冻结骨干）**
   - `freeze=10`，小学习率，只训检测头
5. **第二阶段训练（整体微调）**
   - 解冻全部层，更小学习率继续少量轮数
6. **验证与监控**
   - 同时看**旧类别 + 新类别**精度
   - 旧类别下降 >5% 立即停止，调整数据/学习率

## 六、训练结果判断标准
### 正常收敛
- 损失持续下降，平稳无剧烈震荡
- 新旧类别 mAP@0.5 同步上升或保持稳定

### 异常情况与原因
1. **旧类别精度暴跌**
   - 原因：未混合旧数据 / 学习率太大 / 轮数太多
2. **新类别学不会**
   - 原因：标注混乱 / 增强太强 / 学习率太小
3. **OBB 框不准、角度乱**
   - 原因：标注角度不统一 / 开启 shear / perspective
4. **小目标漏检严重**
   - 原因：漏标 / 分辨率不统一 / 锚框不匹配

---

## 七、最终结论（极简总结）
1. **增量训练成败 = 数据集统一 + 超参数保守**
2. **400 张小样本必须做到**
   - 新旧数据混合训练
   - 标注标准、OBB 角度、标注完整性完全统一
   - 分辨率靠 letterbox 自动统一到 1024
   - 小学习率 + 分层冻结训练，防止遗忘
   - OBB 只开温和增强，关闭剪切、透视

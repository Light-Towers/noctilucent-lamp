# Java 并发与 JVM 深度修行路线图 (2026 版)

## 0. 实验复盘：虚拟线程的“神力”与“死穴” (2026-04-12)
在今天的实验中，我们得出了以下核心结论：
- **神力 (Throughput)**: 开启 1,000,000 个虚拟线程处理 100ms I/O 阻塞任务，仅需 4.8 秒。
- **死穴 (Pinning)**: 使用 `synchronized` 关键字且内部发生阻塞时，会固定 (Pin) 底层载体线程，导致并发度受限于物理核心数（当前机器为 8 核）。
- **锁陷阱 (Serialization)**: 即使使用了 `ReentrantLock`，如果所有线程竞争**同一个**锁，程序依然会退化为串行执行，导致 1000 个 0.5s 任务耗时超过 500s。

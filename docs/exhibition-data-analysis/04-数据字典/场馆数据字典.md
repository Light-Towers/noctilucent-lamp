## 五、数据规则

### 5.1 业务规则

1. **订单金额计算规则**:
   - 订单金额 = 服务单价 × 数量 + 加班费用 - 折扣金额
   - 已支付金额 + 应收金额 = 订单金额 - 已退款金额

2. **状态流转规则**:
   - 订单状态: UNPAY → PAYING → PAYED/REFUNDED
   - 展会状态: STATE_ONE → STATE_TWO → STATE_THREE
   - 审核状态: PENDING → APPROVED/REJECTED

3. **时间规则**:
   - 订单创建时间 ≤ 订单支付时间 ≤ 订单完成时间
   - 展会开始日期 ≤ 展会结束日期
   - 报价有效期 ≥ 当前日期

### 5.2 数据约束

1. **唯一性约束**:
   - 订单号 (order_num) 全局唯一
   - 展会编码 (exhibition_code) 全局唯一
   - 报价单号 (quotation_num) 全局唯一

2. **非空约束**:
   - 所有主键字段不能为空
   - 审计字段 (create_time, update_time) 不能为空
   - 状态字段不能为空

3. **数值约束**:
   - 金额字段必须 ≥ 0
   - 数量字段必须 ≥ 0
   - 百分比字段必须在 0-100 之间
